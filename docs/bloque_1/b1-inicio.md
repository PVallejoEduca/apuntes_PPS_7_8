# 🚀 BLOQUE 1: Los Cimientos (Lenguaje, Entorno y Paradigma)

Bienvenido al desarrollo Android moderno. Olvida lo que creías saber sobre crear aplicaciones. El ecosistema móvil ha evolucionado para ser más rápido, más seguro y, sobre todo, **mucho más divertido de programar**.

En este primer bloque vamos a construir los cimientos absolutos de tu carrera como desarrollador Android. No podemos levantar un rascacielos si el terreno no es firme. Aquí pasaremos de configurar tu primera herramienta de trabajo a entender cómo "piensan" las interfaces reactivas del presente.

!!! success "¿Qué vas a conseguir al final de este bloque?"
    Entenderás cómo compila una app, dominarás la sintaxis moderna de Kotlin, dirás adiós para siempre a los diseños en XML y crearás tus primeras pantallas interactivas que reaccionan en tiempo real a las acciones del usuario.

---

## 🗺️ Tu hoja de ruta

A continuación, tienes los 4 pilares que conforman este bloque. Haz clic en cada uno de ellos para empezar:

### Módulo 0: El Ecosistema Android (Entorno y Compilación)
**Objetivo:** *Entender dónde estamos programando y por qué el proyecto a veces "no compila".*

Antes de correr, hay que saber andar. Aquí instalaremos Android Studio, configuraremos los emuladores sin quemar el ordenador y entenderemos qué son esos archivos `build.gradle` que hacen que todo funcione (o se rompa).

👉 [**Empezar Módulo 0**](modulo_0/b1-m0-inicio.md)

### Módulo 1: Kotlin para supervivientes de Java (Bridge)
**Objetivo:** *Cambiar el chip de Imperativo a Funcional y evitar los `NullPointerException`.*

Si vienes de Java, prepárate para un choque cultural maravilloso. Vamos a quitar todo el código basura (*boilerplate*) y aprenderemos a usar *Lambdas*, *Data Classes* y *Corrutinas* para programar de forma segura y concisa.

👉 [**Empezar Módulo 1**](modulo_1/b1-m1-inicio.md)

### Módulo 2: El Despertar Declarativo (Compose Essentials)
**Objetivo:** *Entender que la UI se "programa", no se "dibuja" en un XML.*

Bienvenido a Jetpack Compose. Aprenderemos a apilar cajas, filas y columnas usando funciones de Kotlin. Descubrirás el poder de los `Modifier` y cómo hacer que tu aplicación sea accesible para todos los usuarios desde el minuto cero.

👉 [**Empezar Módulo 2**](modulo_2/b1-m2-inicio.md)

### Módulo 3: El Estado (State Management)
**Objetivo:** *Controlar la reactividad. Si el dato cambia, la pantalla se actualiza sola.*

El corazón de Compose. Aprenderemos cómo hacer que la interfaz tenga memoria. Si un usuario pulsa un botón, ¿cómo sabe la pantalla que tiene que redibujarse? Aquí dominaremos la "Recomposición" y la elevación del estado (*State Hoisting*).

👉 [**Empezar Módulo 3**](modulo_3/b1-m3-inicio.md)
